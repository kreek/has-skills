# Python

Use this before scaffolding fresh Python projects.

## Defaults

- Package manager: uv. Do not fall back to raw pip/venv because uv is not
  preinstalled. uv consolidates pip, pip-tools, pipx, Poetry, pyenv, and
  virtualenv into a single tool for the whole workflow: there is no need to
  layer any of those on top.
- API default: FastAPI.
- High-throughput alternative: Litestar (msgspec-based serialisation, structured
  controllers) when hot-path serialisation or strict typing throughput matter
  more than FastAPI's ecosystem breadth.
- Robust/full-stack default: Django.
- Avoid raw `http.server` or hand-rolled WSGI/ASGI routing for web apps.

## Choose

- FastAPI for APIs, services, typed request/response models, OpenAPI-first
  development, and lightweight apps.
- Litestar when you want FastAPI-shape ergonomics with faster serialisation,
  first-class DI, and structured controller classes.
- Django when the app needs admin, ORM, auth, forms, templates, migrations, and
  batteries-included conventions.
- Flask only when the user asks, an existing repo uses it, or its ecosystem is a
  better fit for a known constraint.

## Minimum Scaffold

- `pyproject.toml` managed by uv and `uv.lock`.
- Standard commands or task runner aliases for `test`, `lint`, `format`,
  `typecheck`, and app run.
- FastAPI/Litestar/Django app entrypoint generated by the framework, not custom
  HTTP plumbing.
- One behavior smoke test through ASGI/Django test client.
